# ツイッター部隊 運用ワークフロー（Grosmimi Japan）

> 対象アカウント: @grosmimi_japan
> ターゲット: 1歳10ヶ月の娘を持つ日本人ママ（20代後半〜30代）

---

## 部隊構成

| 名前 | 担当 | 実行タイミング | ツール |
|------|------|--------------|--------|
| **企画マン** | 1週間分のツイートプランを生成 → Excelに出力 → Teamsに送信 | 毎週金曜 09:00 JST | `twitter_scheduler.py --generate-weekly` |
| **監督** | 今日の投稿予定と承認状況をTeamsに報告。コメンターの投稿も都度報告 | 毎日 09:00 JST + コメント投稿時 | `twitter_supervisor.py` |
| **ツイート** | 承認済みツイートを自動投稿 | 毎日 10:00・19:00 JST | `twitter_scheduler.py --slot 10/19` |
| **コメンター** | 育児ツイートに共感コメントを投稿（1時間ごとに1人） | 毎日 12:00〜16:00 JST（5回） | `twitter_engage.py --limit 1` |
| **調査マン** | 競合・参考ブランドのTwitter運用を調査してTeamsに報告 | 毎日 16:00 JST | `twitter_research.py` |

## 週次フロー

```
金曜 09:00
  └─ 企画マン: 翌週7日分プランを生成
       ├─ Claudeでツイート生成（カテゴリ5種をローテーション）
       ├─ SharePointにJSONアップロード（7ファイル）
       └─ Excelファイル → Teams送信

金曜〜日曜
  └─ 担当者: ExcelのF列（승인）を入力
       ├─ Confirmed → そのまま投稿
       ├─ Declined + G列に代案テキスト → 代案で投稿
       └─ Declined のみ → スキップ

月〜日 毎日
  ├─ 09:00 監督: 今日の予定と承認状況をTeams報告
  ├─ 10:00 ツイート: 10:00枠を投稿
  ├─ 12〜16時 コメンター: 1時間ごとに共感コメント（計5本）
  └─ 19:00 ツイート: 19:00枠を投稿
```

---

## 日次フロー詳細

### 09:00 — 監督の朝報告（Teams）
- 今日のツイート内容（10:00・19:00）
- 各スロットの承認状況（✅承認済み / ⏳承認待ち / ❌Declined）
- 承認待ちがあれば「承認してください」と促す

### 10:00 / 19:00 — ツイート投稿
1. SharePointから今日のプランJSONをダウンロード
2. ExcelのConfirmed/Declined確認
3. Confirmed → プランのツイートをそのまま投稿
4. Declined + 代案テキストあり → 代案テキストで投稿
5. Declined / 空欄 → スキップ（投稿しない）

### 10:00 / 19:00 — 監督の投稿完了報告（Teams）
- 投稿完了後、監督が日本語でTeamsに通知
- 内容：投稿時間・ツイート本文・投稿URL

### 12:00〜16:00 — コメンター（1時間ごと）
1. Firecrawl（Python SDK）で直近24時間のツイートを検索（1歳10ヶ月向けクエリ）
2. フィルター適用:
   - 直近24時間以内のみ（Snowflake IDでタイムスタンプ確認）
   - 競合・自社アカウント除外
   - 過去に送った相手には送らない
   - 男性・企業アカウントはClaude判定でスキップ
3. Claudeで共感コメントを生成（宣伝なし・50〜80文字）
4. 投稿（in_reply_to → 失敗時は@メンションにフォールバック）
5. 監督がTeamsに結果報告

### 16:00 — 調査マン
1. Firecrawl（Python SDK）で競合・参考ブランド8社のツイートを検索
2. Claudeで各ブランドの投稿傾向・トーン・ハッシュタグ戦略を分析
3. Teamsにテキストレポート送信
4. Excelファイル（`twitter_research_YYYY-MM-DD.xlsx`）を生成してTeamsにアップロード

## 承認ルール（Excel F列）

| F列の値 | G列（代案テキスト） | 動作 |
|---------|-------------------|------|
| `Confirmed` | 不要 | プランのツイートをそのまま投稿 |
| `Declined` | テキストあり | G列のテキストで投稿 |
| `Declined` | 空欄 | スキップ（投稿なし） |
| 空欄 | どちらでも | 投稿なし（承認待ち扱い） |

> **推奨**: 企画マンからExcelが届いたら、その週のうちに承認しておく。
> 当日の投稿時間（01:00 UTC）に承認済みであれば自動投稿される。

## トラブルシューティング

| 問題 | 確認場所 | 対処 |
|------|---------|------|
| ツイートが投稿されない | GitHub Actions ログ | 承認状況・SharePoint接続を確認 |
| Excelがロックエラー（423） | SharePoint | Excelを閉じてから再アップロード |
| コメンターが0件 | Actions ログ | 直近24時間のツイートが検索に引っかからない場合あり（正常）|
| プランJSONが見つからない | SharePoint plans/ フォルダ | 企画マンを手動実行: `--generate-weekly` |
| Twitter API 403エラー | .env ファイル | APIキーの有効期限・権限を確認 |
